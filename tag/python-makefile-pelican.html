Some questions remain

Mon 17 December 2012
By Alex P. Gates

In Blog.

tags: Python Makefile Pelican

Whizzamo!

I managed to get a lot working, but a few questions remain.

Do I need to use a Makefile?

Would it make my life easier?

Now, I am regenerating new HTML by running

pelican . -o ../ -s settings.py

But when I first set up Pelican, I used the quick setup which created a make file

$ pelican-quickstart

That allowed me to publish very easily:

make html

In my attempts to create a Makefile, I managed to delete my entire directory a few times. So, I’m a bit stuck there.

I used fabric, but I don’t know if it is necessary

I did have fab set up, sort of, but I think my fabfile is sloppy:

"""Fabric tasks for publishing a pelican blog to github."""
from __future__ import with_statement

from fabric.api import lcd, local, settings

def pelican():
    """Re-generates the output."""
    local('pelican . -o ../ -s settings.py')

def push(commit_message):
    """Commits the current changes."""
    local('git add .')
    with settings(warn_only=True):
        local('git commit -am "{0}"'.format(commit_message))
    #local('git push origin master')

def publish(commit_message):
    """Re-generates the blog, commits and pushes to github."""
    push(commit_message)
    pelican()
    with lcd('..'):
        local('git add .')
        with settings(warn_only=True):
            local('git commit -am "Pelican autocommit"')
        local('git push origin master')
